# Citation Ledger

Every structural/factual claim this project relies on, with its source. The dataset
itself is synthetic (see DATA_METHODOLOGY.md) — the citations below support the
**schema, field structure, and classification standards** the synthetic data is
modeled on, not specific loan figures.

| Claim | Source URL | Source type | One-line paraphrase |
|---|---|---|---|
| SBA 7(a) loan records include borrower name, lender name, loan amount, approval date, NAICS code, and loan status as standard disclosed fields | https://www.sba.gov/sites/default/files/2018-08/7a_504_FOIA%20Data%20Dictionary.xlsx | Data Dictionary | The official SBA dictionary defines these as the core fields reported for every disclosed 7(a) loan, which this project's schema mirrors |
| The SBA publishes 7(a) loan-level data publicly under FOIA, updated periodically | https://data.sba.gov/en/dataset/7-a-504-foia | FOIA Dataset (Open Data Portal) | SBA's own open-data portal confirms this is a real, periodically-refreshed public release, not a one-off or restricted file |
| Industries are classified using a 2-digit NAICS prefix system (e.g., 31-33 = Manufacturing, 44-45 = Retail Trade, 51 = Information, 62 = Healthcare and Social Assistance) | https://www.census.gov/naics/ | Classification Standard | The U.S. Census Bureau's NAICS system is the official standard this project's 4 sector groupings are mapped to |
| "Charged Off" is a defined loan status category distinct from Active/Paid-in-Full in SBA loan performance reporting | https://www.sba.gov/about-sba/open-government/foia | FOIA Dataset documentation | SBA's FOIA release documentation defines loan status categories, including charge-off, that this project's Status entity type follows |

## What is NOT in this ledger

No row in `public/data.json` (specific loan amounts, specific borrower/lender names,
specific percentages shown in the app's insight panels) is cited as real, because
none of it is. Those figures are generated by `scripts/gen_data.js` using a
deterministic seed, for demonstration purposes only. See DATA_METHODOLOGY.md.
